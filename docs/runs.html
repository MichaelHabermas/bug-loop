    08 / METHODOLOGY

    How to trust this page

    This page never invents a price and never blends unlike runs into one dollar figure. When a run’s cost coverage is incomplete, its cost is shown as a floor (≥) or as n/a with the reason — it is never rounded up to a total and never allowed to win a cost comparison.

    Offline & data handling
    • No network requests. This page and its sibling runs-data.js are the only files it reads.
    • Dropped trace files are parsed in-browser and never uploaded.
    • The only thing stored is your light/dark and Present/Analyze preference.
    • Published runs are schema v1 — recorded before call-level tracking. Missing v2 fields are shown as explicit unknown (v1) markers, never inferred.
    09 / PRESENTER GLOSSARY

    What every number means

    Every specialized word on this page in plain language. Nothing here needs prior knowledge of the pipeline.

    Agent call
    One invocation of an AI worker (triage, test-writer, or fixer). In schema v2 each is recorded with its own usage status; v1 traces predate this.
    Complete cost coverage
    Every agent call in the run has a known dollar figure — reported by the trace or priced from your table. Required before any “cheapest” claim.
    Cost floor (≥)
    The known dollars so far, shown when some calls were never priced. The true cost is at least this much; it is never presented as the total.
    Cost per verified fix
    Known dollars divided by verified fixes. Exact only under complete coverage; otherwise a floor, or n/a with a reason.
    Estimated at your prices
    A dollar value computed for a token-only call whose model matches a price you supplied. Always labelled; never mixed silently into reported totals.
    Fallback
    A recorded moment where an agent could not do the requested thing and a defined backup ran instead. A quality signal, shown in v2 detail.
    Fingerprint
    A stable ID for one failure. Fix events sharing it are attempts on the same incident.
    Frontier
    The lower-left edge of eligible runs: the ones no other eligible run beats on both time and cost.
    Harness
    The surrounding tool that starts an AI worker and reports (or fails to report) its usage — Claude Agent SDK, Codex, or Grok.
    Incident
    One distinct bug. Taken from the dedupe stage’s explicit count.
    Needs human
    An incident routed straight to a person, plus any incident handed off after its allowed fix attempts failed.
    Quality floor
    The reference run’s verified-fix count. A run below it is muted and cannot win a comparison.
    Reference config
    The run every other run is compared against — the baseline configuration when present.
    Resolution source
    Where a model/effort choice came from: a built-in default, an environment variable, or a command-line argument.
    Retry
    A fix attempt after the first for the same fingerprint.
    Seam
    A replaceable pipeline boundary. Spend and time are grouped by triage, test-writing, fix, verify, and other.
    Usage status
    Per call: reported (has dollars), tokens-only (tokens but no dollars), or unavailable (with a reason the harness gave).
    Verified fix
    A verify-stage event whose outcome is verified/pass/passed — the independent check succeeded.
    Wall time
    Elapsed clock time of the whole run, finishedAt minus startedAt.
    Workload key
    Benchmark id, seed, case count, and code revision. Comparisons only mean something within one workload key.
