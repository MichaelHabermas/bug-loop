02 / SEVEN MILESTONES

From one question to a working institution

Each step below is dated and self-contained. Specialized words are underlined — hover or focus them for a plain-language definition, and every one is also collected in the glossary.

  1. Jul 13 — Milestone 1

    The question

    An interview asked a deceptively simple question: “How would you set up your agent pipeline?” The best answer to it is not described — it is demonstrated. The decision made that day: build one for real. Runnable, not rehearsed.

  2. Jul 13, same day — Milestone 2

    Days 1–3 of the plan, shipped in one evening

    First a deliberately buggy service. Then a LangGraphLangGraph is a framework that represents work as connected steps, making saved progress and branching choices explicit. pipeline: read logs, detect real failures, dedupeTo dedupe is to recognize repeated reports of the same failure so one issue is reused instead of opening duplicates., reproduceTo reproduce a bug is to make it fail on demand before editing code, proving a later fix solved the real problem., file GitHub issues, then run the fix⇄verify loop. Three live-run failures taught the real lessons:

    • Dead service. Run against a service that was not answering, and the pipeline correctly refused to fix anything it could not reproduce.
    • Missing installs. WorktreesA worktree is a separate working copy of the Git project, used so an attempted fix cannot disturb the main project folder. without their own dependencies made verification reject three correct fixes rather than ship them unverified. Trust over throughput, by design.
    • Stale branches. Left-over branches were handled by idempotentAn idempotent operation gives the same result whether it runs once or many times, so a retry cannot create a duplicate or a mess. retries.

    Result: the pipeline itself opened pull requests #5–#7, each with before/after evidence — and it correctly declined issue #4, a product-policy ambiguity no automation should decide.

  4. Jul 14 — Milestone 4

    The regression-test institution

    Every fix now ships with a red→green regression testA regression test guards against a bug returning. Red→green means it fails on the buggy code and passes after the fix — direct evidence the behavior improved. written by a separate agent that can touch only tests. This is enforced mechanically: the test must fail on the buggy code and pass after the fix, or the fix does not advance.

    Needs-humanA needs-human incident is one the automation deliberately hands to a person instead of acting on it. incidents get a test.todo instead — a placeholder that never enshrines behavior no human has ratified.

  5. Jul 14 — Milestone 5

    Instrument everything

    A reusable kit was extracted — @bug-loop/core: bring a config and a reproduction adapterA reproduction adapter is a small translator that lets each app describe how to trigger its bug while the pipeline calls one shared interface. plus logs, and it runs. Every run is now tracedA trace is the ordered record of what each step did and why: stage durations, agent calls, tokens, and dollars where the tools report them.: stage durations, agent calls, tokens, and dollars where harnesses report them — never estimated silently. Two dashboards were built.

    Then the sweeps: 10 configurations across two pipelines. The findings were blunt:

    • The framework costs nothing. LangGraph-plus-Grok at 5m18s was within seconds of plain-TypeScript-plus-Grok at 5m15s. The models are the bill, not the plumbing.
    • Cheaper is not cheaper. Haiku burned 5.4× the tokens to save three cents, and retried more.
    • Verification is nearly free. The entire trust layer — the independent check — costs 4–6 seconds, while the agents cost minutes.
  6. Jul 14 — Milestone 6

    Two adversarial design reviews

    Claude and GPT-5.6 reviewed the work as equals. Round 1 redesigned the dashboard around truth-statesA truth-state records how much is actually known about a number — reported, estimated, or unavailable — so nothing is presented as more certain than it is. and a quality-gated cost frontierA cost frontier plots runs by time and dollars so the best trade-offs sit at the lower-left. Only runs with complete cost data and adequate quality may sit on it.. Round 2 tore down the pipeline’s efficiency and found four real problems: a calcifiedCalcification is when an auto-generated test freezes debatable behavior as if it were an intended contract, making future change harder for no ratified reason. auto-generated test pinning debatable behavior as contract, a scope-check hole, a mislabeled trace, and a confounded experiment.

    Corrections ran in both directions — each model fixed things the other raised. The doctrine that came out of it:

    Agents propose, deterministic policy decides, deterministic verification ships.

  7. Jul 15 — Milestone 7

    The doctrine became code

    Routing authority moved out of the agents and into an application-owned tri-state policyA tri-state policy returns one of three answers — deny, authorized, or unknown — so the application, not an agent, decides what is allowed and never guesses.: deny, authorized, or unknown. Regression tests now come from deterministic manifestsA manifest is a fixed, human-authored list describing known bug shapes, so tests for them are generated by rule rather than by an agent. for known bug shapes; agents write tests only for genuinely unknown ones.

    Calcification is now guarded structurally: a positive behavioral pin requires a cited, human-ratified contract before it can exist. Incident workers were made immutableAn immutable worker cannot change shared state, so many can run in parallel without interfering with one another. and parallel-ready.

    The tally: ten agent calls per run became three. The trust guarantees got stronger while the cost went down.

03 / PLAIN LANGUAGE

Every term, defined

The story above uses these words. None of them require prior knowledge of the pipeline.

Adapter
A small translator between one app and the shared pipeline, so each app can trigger its bugs in its own way.
Calcification
When an auto-generated test freezes debatable behavior as if it were an intended contract, blocking change for no ratified reason.
Contract
An agreed data shape and rule between steps, so either implementation can swap in without changing the result.
Cost frontier
Runs plotted by time and dollars; the best trade-offs sit lower-left. Only complete, quality-passing runs may sit on it.
Dedupe
Recognizing repeated reports of the same failure so one issue is reused instead of opening duplicates.
Fixer interface
The agreed input and output for a fixing tool, so Grok or Codex can be swapped without changing the pipeline.
Idempotent
An operation that gives the same result whether it runs once or many times, so a retry cannot create a mess.
Immutable worker
A worker that cannot change shared state, so many can run in parallel without interfering.
LangGraph
A framework that represents work as connected steps, making saved progress and branching choices explicit.
Manifest
A fixed, human-authored list of known bug shapes, so tests for them are generated by rule rather than by an agent.
Needs-human
An incident the automation deliberately hands to a person instead of acting on it.
Observability
The ability to see what a system actually did and why — and to have that be comprehensible to any reasonable person.
red→green
A test that fails on the buggy code and passes after the fix — direct evidence the behavior improved.
Reproduce
Making a bug fail on demand before editing code, to prove the later fix solved the real problem.
Seam
A deliberate boundary where one implementation can be replaced by another without disturbing the system.
Trace
The ordered record of what each step did and why: durations, agent calls, tokens, and dollars where reported.
Tri-state policy
A rule that returns deny, authorized, or unknown — so the application decides what is allowed and never guesses.
Truth-state
A record of how much is actually known about a number: reported, estimated, or unavailable.
Worktree
A separate working copy of a Git project, so an attempted fix cannot disturb the main project folder.

Built in ~2 days of orchestrated agent time by one person directing Claude (Fable 5), GPT-5.6, and Grok 4.5 — the process is the product.
